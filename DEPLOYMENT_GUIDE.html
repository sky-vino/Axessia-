What you will get after completing this guide

By the end of this guide, you will have Axessia's Web Scan Console (WSC) running live on Microsoft Azure at a URL like https://axessia-app.azurewebsites.net. Anyone on your team can open that link in a browser and use the tool immediately — no installation required on their machine.


What gets deployedWhere it runs
Axessia Streamlit frontend (the UI your team sees)Azure App Service — port 8000 (public)
Axessia FastAPI scan engine (the backend that does the scanning)Same App Service — port 8001 (internal only)
AI explanations (Azure OpenAI)Azure OpenAI Service (separate resource)
💡 Only WSC is included. Android and iOS mobile scanning require a physically connected device — they cannot run in the cloud. This package is WSC-only and fully cloud-ready.

PART A — Set up your local computer (one-time)

1

Install Git

Download and install Git from git-scm.com/downloads. Accept all defaults during installation. Git is how your code gets pushed to Azure.

2

Create a GitHub account (if you don't have one)

Go to github.com and sign up for a free account. Remember your username and password.

3

Create a new GitHub repository

On github.com, click the + icon (top right) → New repository. Name it axessia-wsc. Set it to Private. Do NOT tick "Add README". Click Create repository.

4

Create a folder and put your files in it

On your computer, create a new empty folder called axessia-wsc. Copy ALL the files from the ZIP package into this folder. Make sure app.py and startup.sh are at the root (not inside a subfolder).

5

Open a terminal / command prompt in that folder

On Windows: hold Shift and right-click the folder → Open PowerShell window here.
On Mac: right-click the folder → New Terminal at Folder.

6

Run these commands one by one

Copy and paste each line into the terminal and press Enter after each:

# 1. Initialise git in this folder
git init

# 2. Add all files
git add .

# 3. Create the first commit
git commit -m "Initial Axessia WSC deployment"

# 4. Set the main branch name
git branch -M main

# 5. Connect to YOUR GitHub repo
#    Replace YOUR-USERNAME with your GitHub username
git remote add origin https://github.com/YOUR-USERNAME/axessia-wsc.git

# 6. Push the code to GitHub
git push -u origin main

GitHub will ask for your username and password (or a Personal Access Token — see github.com/settings/tokens if prompted).

✅ Your code is now safely on GitHub. Every time you make a change and push, Azure will automatically redeploy.

PART B — Create your Azure OpenAI resource

ℹ️ What is this? Azure OpenAI is Microsoft's version of GPT-4o-mini. It powers the AI explanations in Axessia. You need to create this resource first so you get an API key and endpoint URL.
1

Sign in to Azure Portal

Go to portal.azure.com and sign in with your Microsoft / Azure account.

2

Create a Resource Group

In the search bar at the top, type Resource groups and click it. Click + Create. Fill in:
— Subscription: your subscription
— Resource group name: rg-axessia
— Region: East US (or the closest region to you)
Click Review + Create → Create.

3

Create the Azure OpenAI resource

In the top search bar, type Azure OpenAI and click it. Click + Create. Fill in:
— Subscription: your subscription
— Resource group: rg-axessia
— Region: East US
— Name: axessia-openai
— Pricing tier: Standard S0
Click Next through all tabs, then Create. Wait ~2 minutes.

4

Copy your Key and Endpoint — SAVE THESE

Once deployment is complete, click Go to resource. In the left menu, click Keys and Endpoint. You will see:
— KEY 1 — this is your AZURE_OPENAI_KEY
— Endpoint — this is your AZURE_OPENAI_ENDPOINT (looks like https://axessia-openai.openai.azure.com/)
Copy both to a safe place (Notepad, etc.).

⚠️ Never share your Key or put it in your code files. You will paste it in Azure App Settings — not in any file.
5

Deploy the GPT-4o-mini model

On the same Azure OpenAI resource page, click Go to Azure OpenAI Studio (blue button). In the Studio, click Deployments (left menu) → + Create new deployment. Fill in:
— Select a model: gpt-4o-mini
— Deployment name: gpt-4o-mini
Click Deploy. Wait ~1 minute.

✅ Azure OpenAI is ready. You have your Key, Endpoint, and Deployment name. Keep them handy for Part E.
5

PART C — Create your Azure App Service

1

Search for App Service

In the Azure Portal top search bar, type App Service and click it. Click + Create → Web App.

2

Fill in the Basics tab

FieldWhat to enter
SubscriptionYour Azure subscription
Resource Grouprg-axessia (the one you created)
Nameaxessia-app (becomes axessia-app.azurewebsites.net)
PublishCode
Runtime stackPython 3.11
Operating SystemLinux
RegionEast US (same as OpenAI)
3

Choose the right App Service Plan (size matters!)

Under Pricing plan, click Create new. Name it axessia-plan. Click Explore pricing plans and choose:

B2 (Basic) — 2 CPU cores, 3.5 GB RAM — minimum needed for Playwright + Chromium

⚠️ Do NOT choose B1 or F1 (Free) — Playwright will crash with not enough memory.

💰 B2 costs approximately $27–35 USD per month. You can scale down to B1 only if you are NOT using Playwright scanning (i.e., scanning is disabled).
4

Click Review + Create → Create

Leave all other tabs at their defaults. Click Review + Create. Review the summary. Click Create. Deployment takes about 2 minutes.

✅ Your App Service is created. Your app URL will be https://axessia-app.azurewebsites.net (or whatever name you chose).

PART D — Connect Git to Azure (auto-deploy on push)

1

Go to your App Service

In Azure Portal, go to App Service and click on axessia-app.

2

Open Deployment Center

In the left menu, scroll down to Deployment section and click Deployment Center.

3

Select GitHub as the source

Under Source, select GitHub. Click Authorize and sign in to your GitHub account. Allow the permissions it asks for.

4

Select your repository

Fill in:
— Organization: your GitHub username
— Repository: axessia-wsc
— Branch: main

5

Click Save

Click Save at the top. Azure will immediately start the first deployment. You can watch the progress in the Logs tab below.

📌 From now on: Every time you run git push from your computer, Azure will automatically pull the latest code and redeploy your app within ~3–5 minutes.

⚠️ This step is critical. Without these settings, the app will start but AI features will fail and scans will not work. Do this BEFORE the app finishes its first deployment.
1

Go to Configuration

In your App Service left menu, click Configuration. You will see the Application settings tab.

3

Set the Startup Command

Click the General settings tab (next to Application settings). Find the Startup Command field. Type exactly:

bash startup.sh
4

Click Save

Click Save at the top. Click Continue when it warns the app will restart. This is expected.

5

Enable HTTPS Only

In the left menu, click TLS/SSL settings. Under Protocol Settings, turn HTTPS Only to On. Click Save.

✅ All environment variables are set. Your API keys are stored securely in Azure — never in your code.

PART F — Push your code and watch it deploy

1

Trigger a fresh deployment

Since you've now added all the App Settings, trigger a fresh deployment. In your terminal (in the axessia-wsc folder), run:

# Make a small empty commit to trigger redeploy
git commit --allow-empty -m "Trigger Azure redeploy with app settings"
git push
2

Watch the deployment in Azure

In Azure Portal → your App Service → Deployment Center → Logs tab. You will see a new entry appear. Click on it to see the build log in real-time. The full process takes about 5–8 minutes the first time (Playwright Chromium installs ~150 MB).

You will see lines like:
Installing Playwright Chromium browser...
Starting FastAPI (port 8001)...
Starting Streamlit (port 8000)...

3

For all future changes

Edit your files on your computer, then run:

git add .
git commit -m "describe your change here"
git push

Azure will auto-deploy within 3–5 minutes.

PART G — Test your live app

1

Open your app URL in a browser

Go to https://axessia-app.azurewebsites.net (use your actual app name). The Axessia UI should appear within a few seconds.

2

Run your first scan

Click ➕ Add URL. Type a public website URL (e.g. https://www.bbc.com). Click 🔍 Run Scan. Wait up to 60 seconds. The results dashboard should appear with accessibility scores, charts, and rule breakdowns.

3

Test PDF Export

On a completed scan result, click 📄 Export PDF. A PDF report should download automatically.

4

Verify AI explanations work

Click View Results on any scan. Open any failed rule in the 🔴 Issues tab. You should see AI explanation text (not a generic fallback message). This confirms Azure OpenAI is connected correctly.

Troubleshooting common errors

Error you seeWhat it meansHow to fix it
App loads but scan gives "Cannot connect to API server" FastAPI worker did not start Check App Service Logs (left menu → Log stream). Look for startup.sh errors. Make sure Startup Command is set to bash startup.sh.
AI explanation shows generic fallback text Azure OpenAI settings missing or wrong Go to Configuration and double-check all 4 AZURE_OPENAI_ settings. Make sure the Endpoint ends with /.
App shows 503 Service Unavailable App hasn't finished starting Wait 3–5 more minutes. First startup is slow due to Playwright install. Check Log stream for progress.
Deployment fails with "pip install error" Dependency conflict in requirements.txt Check Deployment Center → Logs for the exact package that failed. Use the exact requirements.txt from this package without editing it.
"Unauthorized" error when scanning AXESSIA_API_KEY mismatch This should not happen in normal use since both processes share the same env var. If it appears, restart the App Service.
Scan times out on large sites Page load > 30 seconds Normal for very heavy sites. Try a specific inner page URL instead of the homepage.
App crashes after a few scans with OOM error Not enough RAM — you are on B1 plan Scale up to B2 in App Service → Scale up (App Service plan) → B2.

🔍 How to check logs: In Azure Portal → App Service → Log stream (left menu, under Monitoring). This shows live output from your app — very useful for debugging startup issues.

🔧 How to SSH into your app (advanced): App Service → Advanced Tools → Go → SSH. This lets you run commands directly inside the container to inspect what's running.
